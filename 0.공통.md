OSI 7 계층|네트워크 통신을 7개의 계층으로 나누어 이해하는 데 사용되는 개념적 프레임워크|각 레이어는 특정 기능을 제공하여 바로 위 및 아래의 레이어와 상호작용을 함. 1. 물리계층: Physical Layer - 장치 간의 물리적 연결을 처리(케이블, 스위치,..이더넷 케이블, 무선신호 등) 2. 데이터 링크 계층: Data Link  - 노드 간 데이터 전송 및 오류 감지/수정(이더넷, 와이파이, ...) 3. 네트워크 계층: Network - 여러 네트워크에 걸쳐 패킷 전달을 관리(IP, ICMP(인터넷 제어 메시지 프로토콜), ... ) 4. 전송계층: Transport - 완전한 데이터 전송, 오류 복구 및 흐름 제어를 보장 5. 세션계층: Session - 애플리케이션 간의 연결을 설정, 유지 및 종료함 6. 표현 계층: Presentation - 애플리케이션 계층과 네트워크 형식 간의 데이터를 변환. 데이터 인코딩, 압축 및 암호화를 처리 (암호화형식 SSL/TLS, 데이터 형식 JPEG/PNG, 인코딩 ASCII/UTF-8 등) 7. 응용 계층: Application - 최종 사용자 애플리케이션에 직접 네트워크 서비스를 제공.  사용자 상호작용이 발생하는 곳(HTTP, DNS, ...)|
HTTPS|웹을 통해 데이터를 전송하는 Hyper Text Transfer Protocol의 확장|암호화를 통해 보안 계층을 추가하여 웹 브라우저와 서버 간에 교환되는 데이터의 보안을 보장함 암호화: SSL또는 TLS와 같은 프로토콜을 사용하여 전송되는 데이터를 암호화. * 웹 서버와 브라우저 사이에 암호화된 링크를 설정하는 보안 프로토콜 인증: 신뢰할 수 있는 인증기관(CA)에서 발급한 디지털 인증서를 통해 사용자가 통신하는 웹 사이트가 합법적인지 등을 확인. *인증서: HTTPS 웹 사이트는 SSL/TLS 인증서를 사용하여 신원을 인증하므로 중간자 공격으로부터 보호|
