Big O Notation|알고리즘의 성능을 측정하는 데 사용되는 수학적 표기법|O(1): 상수시간 - 입력 크기에 상관 없이 일정한 시간이 걸리는 알고리즘(인덱스로 배열 접근) O(log n): 로그 시간 - 입력 크기가 커질수록 걸리는 시간이 느리게 증가(이진탐색) O(n): 선형시간 - 입력 크기만큼 시간이 비례해서 증가(순차 탐색) O(n log n): 선형 로그 시간 - 크기가 큰 데이터를 효율적으로 정렬할 때 사용(퀵 정렬, 병합 정렬) O(n^2): 이차 시간 - 입력 크기가 커질수록 시간 복잡도가 급격히 증가(버블 정렬, 선택 정렬) O(2^n): 지수 시간 - 매우 비효율적인 알고리즘. 입력 크기가 커질수록 실행 시간이 기하급수적으로 증가(피보나치 수열을 재귀로 구현하는 방식) 알고리즘의 효율성을 평가하는 기준|
